# @fragno-dev/ts-trace-cli

`ts-trace` turns raw TypeScript trace output into small, scriptable reports for checker-performance
work.

This package is aimed at workflows like the recent `@fragno-dev/db` type-performance analysis, where
we want to answer questions such as:

- which project files are hottest in `types.json`
- which symbols dominate one file
- whether a refactor removed or reduced a helper type
- whether a win in one hotspot turned into a regression elsewhere

## What TypeScript traces are

TypeScript can emit a trace directory with files such as:

- `types.json`
- `trace.json`

For this MVP, `ts-trace` focuses primarily on `types.json` entry counts. A **count** means the
number of matching entries in `types.json`. It is a useful proxy for checker cost, but it is **not**
a direct timing measurement in milliseconds.

## Generating a trace

From the repo root, run `tsc` with `--generateTrace` against the package you want to inspect.

> `ts-trace` currently expects traces generated by `tsc`. `tsgo` trace generation is not supported.

Example:

```bash
pnpm exec tsc -p packages/fragno-db/tsconfig.json --generateTrace packages/fragno-db/trace-out
```

That produces a directory like `packages/fragno-db/trace-out`.

## Commands

### Summary

Show top files and symbols, including repo-local views by default.

```bash
ts-trace summary packages/fragno-db/trace-out
```

JSON output:

```bash
ts-trace summary packages/fragno-db/trace-out --format json
```

### File

Inspect one repo-relative file and see which declared symbols dominate it.

```bash
ts-trace file packages/fragno-db/trace-out packages/fragno-db/src/schema/create.ts
```

Include sample displays:

```bash
ts-trace file packages/fragno-db/trace-out packages/fragno-db/src/schema/create.ts --samples 10
```

### Symbol

Inspect one symbol across the trace.

```bash
ts-trace symbol packages/fragno-db/trace-out TableBuilder
```

This is useful for questions like:

- did `ColumnsToTuple` disappear?
- did `Table` counts go down?
- is `TableBuilder` still hot?

### Compare

Compare two traces and show file/symbol deltas.

```bash
ts-trace compare ./trace-before ./trace-after
```

This highlights:

- biggest improvements
- biggest regressions
- per-file deltas
- per-symbol deltas

## Filtering

By default, command output is project-local and excludes the usual noise sources:

- `node_modules`
- TypeScript lib declaration files

You can widen or narrow the scope with:

- `--all`
- `--include-node-modules`
- `--include-ts-libs`
- `--include <pattern>`
- `--exclude <pattern>`

Patterns are substring matches by default, and support `*` / `?` glob-style wildcards.

## Fragno-style workflow examples

Validate a suspected improvement in `create.ts`:

```bash
ts-trace file packages/fragno-db/trace-out packages/fragno-db/src/schema/create.ts
```

Check whether a specific helper type is still expensive:

```bash
ts-trace symbol packages/fragno-db/trace-out RefreshTableTargets
```

Compare before/after traces after a refactor:

```bash
ts-trace compare packages/fragno-db/trace-before packages/fragno-db/trace-after
```

Pipe structured output into docs or follow-up tooling:

```bash
ts-trace summary packages/fragno-db/trace-out --format json > trace-summary.json
```
